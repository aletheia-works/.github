# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security problems.

Use GitHub's **Private Vulnerability Reporting** on the affected repository:

1. Navigate to the repository's `Security` tab
2. Click `Report a vulnerability`
3. Fill in the advisory draft

If you cannot use GitHub's flow, email `jambalaya.pyoncafe@gmail.com` with:

- The affected repository and version
- Reproduction steps
- Your assessment of impact

We aim to acknowledge within **7 days** and provide a disclosure timeline within **30 days**.

## Supported Versions

During Phase 0 bootstrap, only the `main` branch of each active repository is supported.
